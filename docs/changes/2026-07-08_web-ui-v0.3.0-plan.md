## 2026-07-08: Web UI v0.3.0 UX 개선 기획 확정

### 사용자 요구 반영 요약
- 현재 Web UI가 파라미터 중심이라 CAD 작업자 관점에서 직관성이 낮다는 피드백 수렴
- 원하는 방향:
  - CAD 불러오기 → 대상/영역 선택 → Gap 조정 → 광원 설정 → Ray Tracing → 결과 확인
  - NX/Creo 느낌의 워크플로우 중심 화면
  - 오른쪽 3D 뷰어 창 + 기본 조작(회전/줌) 제공

### v0 단계 전략
- **v0.3.0**: 화면 구조 재배치 + 라벨/용어 정비 + 간단 모드
- **v0.3.1**: 작업흐름 패널 정비 + 객체/면 선택 구조 반영
- **v0.4.0**: 실제 3D 뷰어 연동(PoC) 시작
- **v0.5.0**: 드래그 선택/영역 오버레이/시나리오 비교

### 이번 실행 범위
1. `docs/web-ui.md`를 v0 UX 목표 문서로 전면 개편
2. 향후 `run_web.py`에서 아래 우선순위로 리뉴얼
   - `CAD / Gap / Emitter / Run / Results` 흐름화
   - 핵심 파라미터만 기본 노출 + Advanced 접기
   - 기본 용어 한글/의도 중심으로 변경
   - 뷰어 레이아웃 뼈대 추가

### 보류 (의도된)
- 실제 파일 업로드 기반의 완전 3D 뷰어 편집
- 드래그/영역선택/리시버 하이라이트 자동 매핑
- CAD 파츠 객체 분할/트리 기반 선택

### 성능/안정성 주의
- v0 단계에서는 계산 로직은 유지하고, UI 경량 변경으로 먼저 검증
- 출력 형식(JSON/CSV/PNG/Report)은 기존 유지
