# Web UI v0.6.10 - Transform Apply/Preview Separation

## Summary
- `Transform` 입력값은 이제 즉시 적용되지 않고 `preview`만 갱신한다.
- 실제 부품 이동/틸트 확정은 오른쪽 `Transform input` 팝업의 `Apply` 또는 `Enter`로만 반영된다.
- 적용이 완료된 부품은 `Full CAD View`에서 원본 대신 빨간색 적용 상태로 유지 표시된다.

## What changed
- `run_web.py`
  - `Transform input` 팝업에 `Apply`, `Reset`, `Restore original` 버튼 추가
  - draft editor state와 committed transform state를 분리
  - summary 문구를 `Applied` / `Preview` 기준으로 구분
  - 적용 완료된 component는 원본 mesh 대신 transformed overlay만 보이도록 표시 로직 조정
  - `Enter` 입력 시 simulation submit 대신 transform apply 우선 동작

## Behavior
- 숫자 입력 중
  - 노란색 preview 유지
  - 아직 실제 transform rule에는 미반영
- `Apply` 클릭 / `Enter`
  - 현재 입력값을 active transform rule에 반영
  - 3D viewer에는 적용 완료 상태만 빨간색으로 유지
- `Reset`
  - 현재 팝업 입력값만 `0`으로 초기화
  - 이미 적용된 위치는 유지
- `Restore original`
  - active component의 적용된 move/tilt를 0으로 되돌림
  - draft 입력값도 함께 원상 복귀

## Notes
- 현재 `Apply` 기준 대상은 `active transform rule`이다.
- 여러 component를 동시에 선택하더라도 popup은 active rule 1개를 기준으로 편집한다.
