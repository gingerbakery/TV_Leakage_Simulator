# Web UI v0.6.7 - Component / Transform Integration

## Why
- `Components`와 `Transform`이 분리되어 있어 실제 CAD 작업 흐름보다 한 단계 더 돌아가야 했다.
- 사용자는 component를 고른 뒤 바로 transform 입력으로 이어지는 흐름을 원했다.
- 좌측 Transform 입력창은 viewer와 중복되어 가독성을 떨어뜨렸다.

## What changed
- `Model import`를 아코디언 메뉴에서 분리하고 상단 고정 카드로 롤백
- 좌측 메뉴 순서를 `ROI 설정 → Components → Transform manager → Material library → Ray tracing → Result`로 재정렬
- `Components` 탭에서 각 component row마다
  - `Transform`
  - `Material`
  버튼 제공
- `Transform` 버튼 클릭 시
  - 해당 component를 선택
  - transform rule 생성/활성화
  - 오른쪽 3D viewer transform 입력창 오픈
- 좌측 `Transform manager`는 입력 UI를 제거하고 rule/요약 표시 중심으로 단순화
- `Copy from selected`, `Mirror X/Y`, `Reset selected`, `Group move` 버튼 제거

## Implementation notes
- `run_web.py`
  - `startTransformForObject(objectId, popupPosition)` 추가
  - `focusMaterialForObject(objectId)` 추가
  - component tree row를 action-button 구조로 변경
  - 좌측 move/tilt 입력 필드를 제거하고 viewer popup/panel만 입력 source로 유지
