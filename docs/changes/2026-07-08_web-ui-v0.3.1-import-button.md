# Web UI v0.3.1 import button

## Summary
- Replaced manual CAD path typing with a Windows file-picker flow in the web UI.
- Added a lightweight upload endpoint so the selected CAD file can be stored locally and loaded immediately.

## UI change
- `Load CAD` step now shows:
  - read-only selected file field
  - `Import CAD` button
  - `Load selected CAD` button
  - `Use sample model` button

## Technical change
- Added `POST /api/upload?filename=...`
- Uploaded files are stored under `_uploads/`
- Supported picker formats:
  - `.obj`
  - `.stl`
  - `.stp`
  - `.step`
  - `.x_t`

## Notes
- STP/X_T still follow the current V1 importer rule and fall back to synthetic geometry.
- This change is about user workflow and file selection convenience, not importer accuracy expansion yet.
