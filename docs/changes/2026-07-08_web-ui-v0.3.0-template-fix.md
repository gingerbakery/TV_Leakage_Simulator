# Web UI v0.3.0 template fix

## Summary
- Fixed a `NameError` in `run_web.py` when opening the v0.3.0 web UI.
- Root cause was an unescaped JavaScript object block inside the Python f-string used to render the page.

## Change
- Escaped the outer braces for `const state = { ... }` in the HTML template builder.

## Impact
- `GET /` no longer crashes with `NameError: name 'mesh' is not defined`.
- The v0.3.0 web page should now render instead of failing on first load.
