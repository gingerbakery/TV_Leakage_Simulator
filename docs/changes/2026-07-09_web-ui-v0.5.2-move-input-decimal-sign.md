# Web UI v0.5.2 move input usability 개선

## 목적
- Gap 이동 입력에서 `-`, `.`, `-.` 같은 중간 입력 상태가 즉시 지워지는 문제를 없애고, 음수 방향 이동과 소수점 입력을 자연스럽게 지원한다.

## 반영 내용
- `run_web.py`의 gap 이동 입력 필드를 `number`에서 `text` + `inputmode="decimal"`로 변경
- 메인 패널과 viewer popup 모두 raw text 입력값을 별도로 유지하도록 수정
- 숫자 파싱은 제출/계산 시점에만 수행하도록 변경
- `-`, `+`, `.`, `-.`, `+.` 입력 중간 상태를 허용하고 내부 계산값은 임시로 `0`으로 처리

## 기대 효과
- `-0.02`, `.15`, `-.05` 같은 값이 끊기지 않고 입력됨
- um 수준의 미세 gap을 mm 소수 형식으로 더 안정적으로 입력 가능
- 메인 입력창과 popup 입력창이 동일한 raw 값 기준으로 동기화됨

## 확인 포인트
- X/Y/Z 칸에 `-0.02`, `.1`, `-.05`를 직접 입력할 수 있는지 확인
- 입력 직후에도 값이 `0`이나 빈칸으로 되돌아가지 않는지 확인
- 실행 시 hidden `gap_move_xyz`에는 숫자 값이 정상 반영되는지 확인
