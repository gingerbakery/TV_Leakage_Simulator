# Web UI v0.5.0 move-only gap UX 통합

## 목적
- gap 생성 방식을 하나의 `객체 선택 -> XYZ 이동 -> gap 생성` 흐름으로 단순화한다.
- 사용자에게 현재 이동량과 추정 gap을 바로 보여준다.
- 실제 mesh 이동 없이도 move preview를 통해 설계 의도를 빠르게 확인할 수 있게 한다.

## 반영 내용
- Gap UI를 move-only 방식으로 통합
  - `X (mm)`
  - `Y (mm)`
  - `Z (mm)`
- gap mode는 UI에서 고정 `component_move_gap`
- 3D viewer에서 객체 클릭 시:
  - 선택 객체 지정
  - 커서 근처 floating move popup 표시
  - 메인 move panel과 입력값 동기화
- 메인 패널 / popup 모두 아래 정보 표시
  - 현재 선택 객체
  - `ΔX`, `ΔY`, `ΔZ`
  - `|Δ|`
  - `Current gap estimate`
- 실제 mesh 이동은 하지 않되, moved position ghost preview를 full view에 오버레이
- web run은 선택 객체가 없을 경우 auto default gap을 만들지 않도록 변경

## 구현 원칙
- 계산은 기존 `component_move_gap` 백엔드 계약을 재사용
- viewer는 preview만 제공하고 실제 geometry는 수정하지 않음

## 기대 효과
- 사용자가 여러 gap 모드를 고민하지 않고 move 개념 하나로 작업 가능
- 현재 이동량 대비 gap 크기를 즉시 확인 가능
- NX/Creo 스타일의 빠른 move 입력 흐름에 가까워짐
