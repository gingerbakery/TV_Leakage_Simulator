# Web UI v0.4.5 ROI 기본 미선택 모드

## 목적
- ROI 선택 방식이 명확히 정해지기 전에는 3D viewer 클릭이 ROI 선택으로 이어지지 않도록 한다.
- 사용자가 실수로 viewer를 클릭해 ROI view로 전환되는 혼란을 줄인다.

## 반영 내용
- `ROI 선택 방식` 기본값을 `선택 방식`(미선택)으로 변경
- 기본 미선택 상태에서는:
  - 3D viewer 클릭 = 하이라이트만
  - ROI 선택 = 없음
  - ROI view 자동 전환 = 없음
- `3D view에서 선택`을 명시적으로 고른 경우에만 viewer 클릭이 ROI 반영으로 동작
- `Component 선택`을 고른 경우에만 component/face 입력 UI 표시
