## 2026-07-08: 웹 UI v0.3.0 CAD-like UX 리뉴얼

### 반영 목적
- 기존 Web UI의 파라미터 나열형 화면을 실제 작업흐름( CAD 로드 → ROI 선택 → Gap 설정 → Emitter 설정 → 실행 )로 정렬.
- 사용자가 보는 부분에서 필요한 버튼/입력만 먼저 노출하고, 고급 설정은 접었다 폈다 할 수 있게 정리.
- CAD 미리보기를 항상 좌측 입력 패널과 분리해 우측에 3D 뷰어를 두는 UX를 적용.

### 실행 변경 사항
- `run_web.py`를 `Web UI v0.3.0`으로 업데이트.
- `GET /api/scene` API 추가:
  - CAD(또는 경로가 비어 있을 때 샘플)에서 메쉬/face/part 정보를 미리 계산해 반환.
  - 객체(부분) 목록, 면 수, 버텍스 수, import note를 제공.
- UI 화면 재구성:
  - Step 1~5 기반의 가이드형 레이아웃 반영.
  - CAD 경로 + `Load CAD` 버튼
  - 객체 체크박스 기반 ROI 설정 + 수동 face index 입력
  - Gap/Emitter/Material/Advanced 구간 분리
  - 우측 3D 뷰어: drag 회전 + wheel 줌 + 선택 face 하이라이트
- 폴더/결과 링크 동작 유지:
  - JSON/CSV/PNG/HTML 링크를 기존 `/outputs/*` 경로로 그대로 제공.

### 남은 과제 (다음 마일스톤)
- 1차 단계에선 x/y/z gap 이동은 UI 안내만 표시(보류), 다음 스프린트에서 기하 변위 기반 Gap 반영으로 연결.
- 객체 선택/ROI 선택과 실제 레시버(face) 추적의 명확도 개선 필요:
  - 대형 CAD에서 객체 분할 정확도 및 Face-id 기준 선택 보조 UI 추가.
- 3D 뷰어 향상:
  - 실제 파트 색상 구분, ray/hit overlay 연동, view reset/fit to geometry 기능.
