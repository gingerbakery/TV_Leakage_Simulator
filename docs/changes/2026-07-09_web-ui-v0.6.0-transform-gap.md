# Web UI v0.6.0 transform gap / tilt 반영

## 목적
- Gap 생성 방식을 `부품 전체 이동`과 `선택 면만 이동`으로 분리하고, 각각에 대해 `move + tilt`를 적용할 수 있도록 정리한다.
- 3D viewer에서 “무엇이 움직이는지”가 더 명확하게 보이도록 preview overlay와 요약 지표를 함께 제공한다.

## 반영 내용
- Web UI 버전을 `v0.6.0`으로 갱신
- Gap 생성 Step을 아래 구조로 재정리
  - `부품 전체 이동 (기본)`
  - `선택 면만 이동`
- `X/Y/Z 이동(mm)` + `Rx/Ry/Rz 기울기(deg)` 입력 추가
- `선택 면만 이동` 모드는 현재 ROI face 집합 + 추가 face index 입력을 local target으로 사용
- preview overlay on/off 체크박스 추가
- 3D viewer preview를 `translation + rotation` 기준 ghost overlay로 갱신
- summary에 아래 정보 추가
  - 대상 이름 또는 face count
  - move vector
  - tilt vector
  - 예상 최대 상대 이격(mm)

## 백엔드 변경
- `GapRule.rotation_vector_deg` 추가
- `gap.py`에서 target face별 gap strength를 아래 기준으로 계산
  - normal 방향 translation 성분
  - pivot 기준 rotation displacement 성분
- `component_move_gap`은 component 전체 face 집합
- `face_gap`은 local face set에 대해 동일 transform 로직 사용

## 검증
- `py_compile` 통과
- synthetic scene 기준 `component_move_gap + tilt` 실행 확인
- synthetic scene 기준 `face_gap + tilt` 실행 확인

## 후속 메모
- 현재 tilt는 rigid rotation 근사이며, 실제 탄성 휨 해석은 아님
- 추후 `edge bow`, `corner lift`, `center sag` 같은 전용 휨 모드는 별도 backlog로 확장 가능
