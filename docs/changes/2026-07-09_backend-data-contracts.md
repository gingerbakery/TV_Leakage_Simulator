# 2026-07-09 백엔드 데이터 계약 문서 정리

## 목적
- ROI, Gap, Ray trace 개발자가 병렬 작업해도 충돌하지 않도록 백엔드 책임 경계와 데이터 계약을 문서로 고정했다.

## 반영 내용
- `docs/backend-data-contracts.md` 추가 및 기준 문서화
- ROI / Gap / Ray trace / Engine의 책임 범위 명시
- 현재 V1 데이터 흐름 계약 정리
- 추후 확장용 권장 계약안 정리:
  - `ROISelectionResult`
  - 확장형 `GapRule`
- 프론트엔드와 백엔드의 분리 정책 명시

## 핵심 결정
- ROI가 Gap, Ray trace에 영향을 주는 것은 허용한다.
- 단, 영향은 내부 구현 공유가 아니라 `계약 데이터`를 통해서만 전달한다.
- ROI 백엔드 분리는 필수이며, ROI UI 완전 독립은 현재 단계에서 필수 아님.

## 후속 운영 원칙
- 공용 타입 변경 시 반드시 `docs/changes/*.md`에 기록
- 계약 변경은 구현보다 먼저 문서 반영
- 기능 담당 개발자는 자신의 소유 모듈과 계약 경계만 우선 준수
